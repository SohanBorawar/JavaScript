  • What is the difference between ViewState and SessionState?

    • State management concerned with the client-side uses View state. It stores the information in text fields, checkboxes, and radio buttons within a hidden field of a web page, not shared between the other web pages of the website.
    • a POST event, it'll retain this information only accessible by elements of the same web page; however, loss of this information occurs as soon as another page is loaded.

    • Server-side state management utilizes user sessions to maintain information and its flow shared between the website's web pages. The web server stores the client session information and uses cookies to store keys associated with a particular client.
    • Session expiration can clear a user's information. A timeout occurs due to a user's inactivity on a webpage, and the session expires to prevent data loss to non-authentic users.

  • What are the disadvantages of using innerHTML in JavaScript?

  • There are several disadvantages to using innerHTML in JavaScript, including:

    1. Security risk: innerHTML can be used to inject malicious code into a web page, which can then be executed by the user's browser. This is known as a cross-site scripting (XSS) attack.
    2. Performance: innerHTML is slower than other methods of DOM manipulation, such as document.createElement() and document.createTextNode(). This is because innerHTML requires the entire HTML document to be re-parsed.
    3. Lack of validation: innerHTML does not validate the HTML that is passed to it. This can lead to errors in the DOM, which can then cause the web page to malfunction.
